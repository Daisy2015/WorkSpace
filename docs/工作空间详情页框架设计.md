## 三、交互逻辑设计

### 3.1 卡片展开/折叠逻辑

```typescript
// LoopCard 折叠状态管理
interface LoopCardState {
  isExpanded: boolean;           // 卡片是否展开
  expandedActionId: string | null; // 当前展开的 Action（工具链）
}

// 默认状态
- UserMessageCard: 始终展开
- LeaderCard: 默认展开，可折叠
- PositionAgentCard: 默认展开，不可折叠
- LoopCard: 默认折叠，点击展开
- StageResultCard: 始终展开
- FinalResultCard: 始终展开

// 展开规则
1. 点击 LoopCard 标题 → 展开/折叠整个卡片
2. 点击 Action 中的通用智能体 → 展开/折叠工具链
3. 同一时间只能展开一个工具链
4. 展开动画：150ms ease-in-out
```

### 3.2 滚动行为

```typescript
// 自动滚动策略
1. 新消息到达 → 自动滚动到底部（smooth）
2. 用户手动上滚 → 禁用自动滚动
3. 执行中状态 → 锁定在最新卡片
4. 点击资源标签 → 左侧面板联动，不影响滚动

// 滚动性能优化
- 虚拟滚动（卡片数量 > 50 时启用）
- 懒加载（超出视口的卡片延迟渲染）
- 滚动节流（16ms）
```

### 3.3 状态同步

```typescript
// 执行状态流
User Input → Leader (running)
          → Position Agent (running)
          → Scene Agents (running → completed)
          → Final Result (completed)

// 状态标签
- waiting: Gray-100/600 "等待中"
- running: Blue-100/700 "执行中" + 进度条
- completed: Green-100/700 "已完成"
- error: Red-100/700 "异常"
```

### 8.2 状态管理

// 使用 Context + Reducer
const WorkspaceContext = createContext();

const workspaceReducer = (state, action) => {
  switch (action.type) {
    case 'ADD_MESSAGE':
      return { ...state, messages: [...state.messages, action.payload] };
    case 'UPDATE_AGENT_STATUS':
      return { ...state, agents: updateAgentStatus(state.agents, action.payload) };
    case 'TOGGLE_LOOP_CARD':
      return { ...state, expandedLoops: toggleExpanded(state.expandedLoops, action.payload) };
    default:
      return state;
  }
};

## 十四、关键交互流程

### 14.1 用户提问 → 完整执行

```
1. 用户在 ChatInputBar 输入问题
   ↓
2. 点击发送（或 Enter）
   ↓
3. ConversationArea 添加 UserMessageCard
   ↓
4. 发送请求到后端
   ↓
5. WebSocket 推送 Leader 开始执行
   ↓
6. 添加 LeaderCard（status: running）
   ↓
7. Leader 调度岗位智能体
   ↓
8. 添加 PositionAgentCard（显示拆解的场景）
   ↓
9. 场景智能体依次执行
   ↓
10. 每个场景添加 LoopCard（默认折叠）
   ↓
11. 关键节点插入 StageResultCard
   ↓
12. 所有场景完成后，添加 FinalResultCard
   ↓
13. 更新右侧 AgentCenter（显示执行历史）
```

### 14.2 资源引用流程

```
1. 用户点击 ChatInputBar 的 "🔗" 按钮
   ↓
2. 弹出资源选择器（Popover）
   ↓
3. 选择"从左侧选择"
   ↓
4. 左侧 FactsPanel 高亮可选资源
   ↓
5. 点击资源 → 添加到输入栏（Chip 形式）
   ↓
6. 发送消息时携带资源 ID
   ↓
7. 在 UserMessageCard 中显示资源标签
```

### 14.3 工具链展开流程

```
1. 用户点击 LoopCard 标题
   ↓
2. 展开卡片（显示 T→A→O→P）
   ↓
3. 点击 Action 中的通用智能体按钮
   ↓
4. 展开工具链（缩进显示 4 个工具）
   ↓
5. 再次点击 → 收起工具链
   ↓
6. 点击其他通用智能体 → 自动收起当前，展开新的
```
