# JuraWorkSpace 工作空间详情页 — 四版本产品迭代 PRD + 页面功能设计

> 目标页面：基于 **左侧资源范围 + 中间问答工作区 + 右侧嵌入智能体能力区** 的统一工作空间。
> 核心策略：**页面骨架保持一致，按版本动态加载右侧能力层、协同模式、自动化深度。**

---

# 1. 产品版本总体策略

## 1.1 统一设计原则

- **左侧资源区统一**：所有版本共享 MBU 边界、对象实例、资源树
- **中间工作区渐进增强**：从问答 → 任务闭环 → 岗位接管 → 系统自治
- **右侧能力区版本差异化最强**：作为商业化升级的核心感知区
- **升级可视化明显**：未购买能力显示「能力卡占位 + 升级引导」
- **所有高版本能力均叠加低版本能力**

# 2. 四版本产品定义（PRD）

---

# V1 基础版 Foundation

## 2.1 产品定位

**成熟度：L1–L2（会答、会写）**

聚焦“知识理解 + 初稿生成”，让一线人员快速完成信息获取与文档起草。

## 2.2 目标用户

- 一线业务人员
- 研究人员
- 新员工
- 临时项目支持人员

## 2.3 核心能力

### 左侧
- 资源树选择
- 多文档范围限定
- 对象实例切换

### 中间
- 自然语言问数
- 文档问答
- 知识检索增强回答
- 智能摘要
- 报告/PPT 初稿生成

### 右侧嵌入能力（仅基础能力）
- JuraReport（报告初稿）
- JuraPPT（汇报材料初稿）
- SmartSummary（摘要助手）

## 2.4 页面设计重点

### 右侧区域
采用 **3 卡片轻量布局**：

- 报告助手
- PPT 助手
- 摘要助手

下方显示最近生成记录。

## 2.5 核心价值

- 降低认知门槛
- 资料查找时间减少 80%
- 初稿编写标准化

# V2 专业版 Professional

## 3.1 产品定位

**成熟度：L3（会做一件事）**

实现单场景任务闭环：取数 → 分析 → 输出。

## 3.2 目标用户

- 地质工程师
- 油藏工程师
- 钻完井分析人员
- 场景负责人

### 叠加基础版全部能力

### 新增右侧能力层
- 异常诊断 Agent
- 方案对比 Agent
- 参数优化 Agent
- JuraAgent（工具编排）
- GeoMapX（标准图件）

### 中间新增
- 闭环任务执行模式
- SQL / Python / 图件流程可视化
- 输出物模板化

## 3.4 页面设计重点

### 右侧能力布局升级为两层

## 第一层：通用生产工具
- JuraDiagX
- JuraAgent
- GeoMapX
- JuraReport
- JuraPPT

## 第二层：场景模板区
- 异常诊断
- 方案优选
- 井间对比
- 趋势预测

### 中间新增“任务执行轨迹”
展示：
- 取数
- 调用工具
- 生成结论
- 输出图件

## 3.5 核心价值

- 从建议升级为结果
- 缩短专业分析周期
- 减少人工切换多系统

# V3 岗位代理版 Enterprise

## 4.1 产品定位

**成熟度：L4（会干一类活）**

以岗位职责为中心，形成“数字员工工作台”。

## 4.2 目标用户

- 钻完井管理岗
- 动态分析岗
- 地质研究岗
- 项目管理岗

## 4.3 核心能力

### 叠加专业版全部能力

### 新增岗位级能力
- 岗位专属 Agent（按角色装配）
- 任务 SOP 模板
- 长任务记忆
- 多轮版本管理
- 人工确认节点
- 自动日报 / 周报
- 异常触发待办

### 运行时能力
- JuraClaw Runtime
- 上下文记忆
- 长链路任务恢复
- 可审计日志

## 4.4 页面设计重点

### 页面结构从“问答页”升级为“岗位工作台”

右侧区域变为 **岗位能力面板 + 今日待办**：

#### 岗位能力
- 生产分析岗 Agent
- 异常巡检岗 Agent
- 报表值守岗 Agent

#### 今日自动任务
- 待确认异常 3
- 待发送日报 1
- 图件待审批 2

### 中间区域新增 Tab
- 对话
- 自动任务
- 历史版本
- 审批记录

## 4.5 核心价值

- 系统性接管岗位职责
- 沉淀专家经验
- 重复性工作自动化
- 实现岗位代理

# 6. 页面版本差异化设计（核心）

## 6.1 页面差异矩阵

| 区域 | 基础版 | 专业版 | 岗位代理版 | 旗舰版 |
|---|---|---|---|---|
| 左侧资源树 | 标准 | 标准 | 标准+SOP资源 | 多空间联动 |
| 中间工作区 | 问答 | 闭环执行 | 岗位任务台 | 协同驾驶舱 |
| 右侧能力区 | 通用工具 | 场景Agent | 岗位Agent | Agent生态 |
| 历史记录 | 对话记录 | 任务记录 | 版本+审批 | 团队协同历史 |
| 自动化程度 | 低 | 中 | 高 | 极高 |

# 7. 商业化升级设计建议

## 7.1 页面内升级触点

建议在右侧能力区使用：

- 灰态能力卡
- 「旗舰版可用」标签
- 点击后查看升级价值
- 展示预计节省时长

例如：

> 多井批量分析（旗舰版）
> 预计节省 6 小时 / 天

## 7.2 升级路径设计

基础版用户进入页面：
- 看见 GeoMapX 灰态
- 提示：升级专业版即可自动出图

专业版用户：
- 看见 岗位代理卡片
- 提示：升级岗位代理版实现自动日报

# 8. 推荐研发优先级（版本路线）

## Phase 1
Foundation → Professional

优先做出最强商业闭环：
- 问答
- 报告
- 图件
- 异常诊断

## Phase 2
Professional → Enterprise

打造岗位数字员工：
- SOP
- 长任务
- 人工确认
- 版本管理

## Phase 3
Enterprise → Flagship

升级组织级自治：
- Multi-Agent
- 多井协同
- 驾驶舱
- 经营联动

---

# 9. 最终设计建议（一句话）

> **不要做四套页面，而是做“一套工作空间骨架 + 右侧能力插槽 + 中间模式渐进增强”的版本化产品体系。**

这样最利于：
- 商业化升级
- 前端组件复用
- 用户学习成本降低
- 企业级权限控制
- 后续 Agent Marketplace 扩展
