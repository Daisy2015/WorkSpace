# NER 模型后台管理页面功能方案设计（后台管理工作台）

## 1. 页面目标

构建一个面向 **算法工程师 / AI产品经理 / 运营管理员** 的 **NER 模型管理后台页面**，实现从模型训练、状态监控、版本切换、配置查看、在线测试到模型治理的完整闭环。

核心目标：

- 统一管理 NER 模型训练记录
- 实时查看模型训练状态
- 快速切换当前线上生效模型
- 支持模型测试验证识别效果
- 支持模型配置查看与删除治理
- 为后续 A/B Test、灰度发布、效果评估预留能力

***

# 2. 页面整体结构

建议采用 **左右分栏 + 顶部操作区** 的经典 AI 运维后台布局。

## 2.1 顶部操作区

### 功能模块

1. **训练新模型（主按钮）**
   - 点击后立即创建一条新的训练任务记录
   - 自动生成模型版本号（如：NER\_v20260330\_001）
   - 写入任务队列并进入训练中状态
2. **当前生效模型展示**
   - 显示当前线上使用模型名称
   - 展示发布时间、准确率、更新时间
3. **筛选区**
   - 按训练状态筛选：
     - 全部
     - 训练中
     - 成功
     - 失败
     - 已上线
   - 按时间筛选
   - 按模型版本搜索

# 3. 核心功能设计

## 3.1 新建训练记录逻辑

### 交互逻辑

点击 **【训练新模型】**：

1. 创建新的训练记录
2. 自动插入列表第一行
3. 状态初始为：**排队中**
4. 进入异步训练任务
5. 前端通过 WebSocket / SSE 实时刷新状态

### 状态流转

```
排队中 → 数据准备 → 训练中 → 评估中 → 成功/失败

```

### 实时展示内容

- 当前 Epoch：3/10
- 当前 Loss：0.124
- 当前 F1：92.3%
- 预计剩余时间：2min

建议使用：

- 状态 Tag
- 进度条
- 实时日志抽屉

***

## 3.2 在线状态展示

建议使用 **状态列 + 右侧日志抽屉**。

### 状态颜色规范

- 灰色：排队中
- 蓝色：训练中
- 绿色：成功
- 红色：失败
- 紫色：当前线上模型

### 实时刷新机制

推荐：

- **SSE（推荐）**：轻量单向推送
- WebSocket：适合高频日志
- 轮询兜底：5 秒一次

### 训练日志信息

点击状态可展开：

- 数据加载完成
- 标签映射完成
- 模型初始化完成
- Epoch 训练日志
- 验证集指标
- 模型保存路径

***

## 3.3 设置当前使用模型

### 操作入口

每条成功模型记录提供按钮：\
**【设为当前模型】**

### 交互逻辑

1. 点击后弹窗二次确认
2. 展示：
   - 当前线上模型
   - 待切换模型
   - 两者 F1 对比
3. 确认后：
   - 当前记录标记“已上线”
   - 旧模型自动降级为“历史版本”
   - 更新推理服务配置

### 页面展示

当前模型增加：

- 高亮边框
- “当前使用”标签
- 顶部同步展示

***

## 3.4 删除模型

### 删除规则

仅允许删除：

- 非当前使用模型
- 状态成功或失败
- 无关联线上实验任务

### 删除弹窗

展示：

- 模型名称
- 训练时间
- 指标信息
- 是否同时删除模型文件

### 删除后效果

- 从列表移除
- 写入审计日志
- 支持回收站（推荐）

***

- Precision
- Recall

# 6. NER训练数据集管理页面方案设计（语料管理工作台）

## 6.1 页面目标

构建一个面向 **算法工程师 / 标注运营 / AI产品经理** 的 **NER训练语料管理页面**，统一管理训练语料模板，并支持通过“变量插入 + 自动展开”的方式，快速生成批量高质量训练样本。

核心价值：

- 统一维护训练语料模板
- 支持新增、删除、批量导入
- 支持语料模板中插入变量
- 基于变量字典自动生成多条语料
- 降低人工编写重复样本成本
- 提升实体覆盖度与场景泛化能力

***

## 6.2 页面结构

建议采用 **左侧变量池 + 中间语料模板列表 + 右侧生成预览** 的三栏结构。

### 顶部操作区

- **新增语料模板**
- 批量导入
- 批量删除
- 一键生成训练集
- 按实体类型筛选
- 按数据来源筛选

***

## 6.3 语料模板管理列表

采用高信息密度表格。

字段

说明

模板名称

语料模板标题

原始模板

支持变量占位的文本

变量数量

当前模板引用变量数

可生成条数

基于变量组合后的样本量

标签类型

涉及实体类型

更新时间

最近编辑时间

操作

编辑 / 删除 / 预览生成

示例模板：

```
{井名}在{工序}阶段使用{工具}进行施工。

```

***

## 6.4 新增语料（重点支持插入变量）

点击 **【新增语料模板】** 打开编辑弹窗。

### 上方：语料编辑区

支持自然语言输入，并可在光标位置插入变量。

你截图中的字段建议直接作为 **快捷变量插入区**：

- 对象名称
- 对象类型
- 组件名称
- 成果名称
- 数据项类型
- **插入变量 ▼**（下拉更多）

交互方式：

1. 用户输入基础句式
2. 点击字段标签
3. 自动插入变量占位符
4. 形成动态语料模板

示例：

```
{对象名称}的{组件名称}产生了{成果名称}。

```

***

## 6.5 变量池管理

左侧提供 **变量字典池**。

### 数据结构建议

每个变量绑定一组可枚举值：

```
{
  "对象名称": ["A1井", "B2井", "C3井"],
  "组件名称": ["套管", "钻头", "井口装置"],
  "成果名称": ["测井曲线", "录井报告"]
}

```

支持：

- 新增变量
- 编辑变量值集合
- Excel导入变量值
- 绑定知识库实体池
- 按业务对象自动推荐变量值

***

## 6.6 动态生成一组语料（核心能力）

系统基于模板中的变量占位符，自动做笛卡尔组合生成训练样本。

### 生成逻辑

模板：

```
{对象名称}使用{组件名称}生成{成果名称}

```

变量值：

- 对象名称：A1井、B2井
- 组件名称：钻头、套管
- 成果名称：测井曲线、录井报告

自动生成：

```
A1井使用钻头生成测井曲线
A1井使用钻头生成录井报告
A1井使用套管生成测井曲线
...

```

即可由 **1条模板动态生成 N 条训练语料**。

建议支持：

- 全组合生成
- 随机采样生成
- 去重生成
- 按规则过滤非法组合

***

## 6.8 删除语料

- 删除已生成样本

删除模板时可选：

- 是否同步删除已生成语料
- 是否保留历史训练版本引用
